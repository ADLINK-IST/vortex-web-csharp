Get Started

Create a Vortex Object

To connect to the Vortex data sharing platform you need a Vortex object. It requires knowing the id of the domain you are connecting to. In the example below we connect to domain 0 (zero).

var domainId = 0;
Vortex vortex = new VortexImpl (domainId);

Connect to Vortex

To establish a connection to Vortex you need the URL of the server and an authentication token.

The URL is for the Vortex Web server you are connecting to. Running the server on the same machine as the application the URL will be http://localhost:9000. The default port used by the Vortex Web server is 9000.

The authentication token is dependent on the access management pluglin you are using. By default there is no access management, the token can be any string.


var URL = "http://localhost:9000";
var Token = "nopwd";
await vortex.Connect (URL, Token);

Create a Topic

In Vortex to read or write data you first need to register a Topic. A topic is registered with a unique name and a type. Default quality of service settings for readers and writers of the topic data can also be provided.

The topic name must be unique within the domain you are connected to. If registering a topic that may have already been registered then it must register with the same type.

The type of the topic is provided in two pieces (a) a programming language specific type and (b) a name used for registering the type. The programming language specific type will point to a class in Java. The name used for registering is the name of the type that is by Vortex.


var topicName = "TChatMessage";
var topicType = "com.prismtech.cafe.demo.idl.ChatMessage";
var topicRegistrationType = "com.prismtech.cafe.demo.idl.ChatMessage";
var qos = new List<QosPolicy>();

var topic = await vortex.CreateTopic<ChatMessage> (topicName, 
                                                topicType, 
                                                topicRegistrationType, 
                                                qos);

Read Data

To read Vortex data you need a data reader. A data reader is created for a specific topic and quality of service. The topic is one that is registered with Vortex. The quality of service is an optionally empty list of quality of service policies.


var qos = new List<QosPolicy>();
var reader = await vortex.CreateDataReader<ChatMessage> (topic, qos);

reader.OnDataAvailable += (object sender, SampleData<ChatMessage> e) => {
    Console.WriteLine("Received: " + e.Data);
};

Write Data

To write Vortex data you need a data writer. A data writer is created for a specific topic and quality of service. The topic is one that is registered with Vortex. The quality of service is an optionally empty list of quality of service policies.


var qos = new List<QosPolicy>();
var writer = await vortex.CreateDataWriter<ChatMessage> (topic, qos);

var msg = new ChatMessage ("csharp", "roks!");
for (var i = 0; i < 10; i++) {
    await writer.Write (msg);
}

Disconnect

When you have finished reading or writing data it is a good idea to close the connnection to Vortex.


await vortex.Disconnect ();
await vortex.Close ();
